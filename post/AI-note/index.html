人工智慧 Artificial Intelligent

  • 圖靈測試：測試機器是否能表現出與人等價或是無法區分的智慧
  • 赫布理論(Hebbian Theory)：解釋了在學習的過程中腦中的神經元所發生的變化
  • 賽局理論：研究遊戲中個體與團體如何做決策與如何找到最佳決策的一門學科
  • 納許均衡：囚徒困境，個人最佳選擇與團體最佳選擇衝突
  • 機器翻譯：Zero Shot 語言之間的連結
  • 通用人工智慧：整合各能力

機器學習 Machine Learning

機器能從經驗或資料中學習，而後在某個問題或任務上衡量標準能有所提升
三大要素

  • 模型
  • 算法
  • 策略

學習類型

學習類型 Learning type 訓練資料 目標 舉例
監督式學習 Supervised Learning 資料、標籤 學習資料與標籤之間的關係 線性回歸(Linear Regression)
非監督式學習 Unsupervised Learning 資料 學習資料之間的隱藏結構 分群(Clustering)
強化式學習 Reinforcement Learning 來自一系列與環境互動的行動與獎勵回饋(reward) 學習如何選擇動作並累績最大的獎勵

機器學習流程

訓練資料的問題

  • 噪音(Noise)
  • 偏誤(Bias)
  • 獨立性

線性模型特徵轉換

  • 線性函數(Linear model)：f(x)=w1x1+w2x2+...+wnxn=wTxf(x) = w_1x_1 + w_2x_2 + ... + w_nx_n = w^Txf(x)=w1​x1​+w2​x2​+...+wn​xn​=wTx
  • 線性假設空間(Linear hypothesis)
  • 特徵轉換(Feature Transform)
    • 把非線性問題線性化
    • 把資料轉換到新空間，再用線性模型處理

過擬合(Overfitting)

  • 預測模型在訓練資料上表現好，不代表在真實資料上表現好
  • 模型容易受資料影響
    • 簡單模型優先
  • 假設空間太大 太複雜
    • 限制假設空間能挑的範圍
  • 有噪音或變異很大
    • 清理或預處理

正則化(Regularization)

  • 避免資料太過複雜

訓練神經網路困難點

區域最小值(Local minimum)

  • 有很多區域最小值不一定是全局最小值
  • 當隱藏層越深 最後區域最小值變異較小 幾乎等同於全局最小值

鞍點(Saddle Points)

  • 不是極值但梯度最小
  • 隨機梯度與批次梯度演算法帶有 noise 擾動可逃脫鞍點得到全局最小

梯度消失(Vanishing Gradint)

  • 反向計算經過激活函數 SigmoidSigmoidSigmoid 微分後的值無梯度 最淺層無改進資訊
  • 用 ReluReluRelu 較容易避免，將資訊帶到深層幫助訓練神經網路

幫助訓練神經網路的方法

Dropout

  • Data 太複雜
  • 隨機拿掉一些點去訓練 把模型複雜度降低 正則化
  • 根據機率組合成期望狀態的神經網路

Batch Normalization

  • Data 差異大 對神經元來說不易學習
  • 將 data 正則化 調整差異
  • 降低每個 data 對模型的影響力

深度學習 Deep Learning

特徵(Features)

  • 為一個函數，能擷取出輸入的某些特性
  • 好的特徵要有的特性
    • 要夠具有代表性
    • 要能容忍變異
  • 層次結構組合成更複雜的特徵

特徵學習

  • 一組特定的權重值，代表從資料所學習到的模式
  • 自動學習特徵：支持向量機SVM(support vector machine)

表徵(Representation)

抽出真正的概念 可以操作 想像 且是有意義的

  • 心智表徵：是假設性的，能夠表示外在現實的內在認知象徵，或者是一種可以讓心智過程得以利用的內在認知象徵
  • 資料表徵:以不同的基底來重新表示資料
  • 語意表徵：Google Zero Shot

表徵要能維持資料原始的特性

  • 非分散式表徵：貓、狗
  • 分散式表徵 字詞之間的關係 可用向量內積算關係

表徵學習

  • 深度學習
  • 主成份分析PCA(Principal components analysis)
  • 自動編碼機(Autoencoder)
    • 將輸入資料壓縮編碼成表徵，而該表徵能被解碼還原成接近原始的資料
    • 學習到的表徵需要隱含足夠的輸入資料的訊息
